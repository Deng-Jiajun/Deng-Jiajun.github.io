语雀试用小结

作为不务正业专业户，非资深markdown程序员，老是花里胡哨瞎捣鼓的三分钟热度患者的我。对各种博客、笔记软件总想去试试。印象笔记、有道云笔记、微云笔记、微信内置收藏笔记、QQ内置收藏笔记、onenote、notion、baklib、各类静态博客、动态博客……甭管专业不专业的，能记录，能写字的地方基本都用过/见过/具体了解过（当然实际上记录没两个字，就瞎玩）
目前稳定使用的是 GitHub Pages +Markdown
语雀自然也听说过，没仔细了解，最近看到有人安利语雀markdown使用体验不错，今天简短地试用了一下，写一篇小结，记录一下关键点。
成文日期：2020年6月22日

1. Markdown语法不可逆

文章编写中使用的markdown语法，生效后产生对应的元素，无法再对起进行编辑。就是说markdown语法在语雀中等价于快捷键，输出就生效，不能再去改markdown本身。
例如：输入 ### 三级标题 ，输入三个 # + 空格 之后，当前行的内容就具有了三级标题的属性。只能通过其他方式去修改为二级标题，而不能直接用 ← 键退回去，删掉一个 # 。
总而言之：语雀的编辑页面就是一个没有源代码模式，且在当前行无法显示源代码的Typora。作为使用人数（应该或许可能）最多的markdown编辑器Typora的使用者而言，这点（至少对我而言）很要命。

2. Markdown导入导出

markdown导入导出自然是有损的，不需要多说

3. 没有图片输入语法

图片![]() 语法不可用
由于语法不可用，导致了一个新的问题：**无法直接使用第三方的图片。不论何种方式，**在文本中插入的(可访问的)图片都会被上传到语雀的服务器。
场景示例1：在语雀编辑器中插入图片，图片将直接上传至语雀服务器。
场景示例2：在typora等markdown编辑器里编辑好，里面的图片链接是互联网可访问（如： ![typora官网图片](``[https://typora.io/img/new/image.png](https://typora.io/img/new/image.png)``)）。直接将.md文件导入，语雀在转换时会将其中的图片上传到语雀服务器。
场景示例3：在typora等markdown编辑器里编辑好，里面的图片链接是本地链接（如： ![image](/img/image.png)）。直接将.md文件导入，语雀在转换时无法识别，将直接引用。

查看markdown或者html可以发现，语雀直接就照着填了👇

4. 桌面端

各项功能还在做…
有一个疑问就是快捷键

Typora中我们习惯的快捷键都是 ctrl + 数字键 设置标题，这里使用的是 alt + ctrl + 数字键 
不知道出于什么考虑要加上这么一个 alt 而且目前不能自己修改快捷键

5.会员体系

语雀目前给出的付费方案是按流量付费。普通用户终身10G，会员首次付费赠送90G，额外购买10元/G。
