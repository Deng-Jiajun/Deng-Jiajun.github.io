Hugo + Github Pages 搭建Blog

本文简单记录通过 Hugo + Github Pages 搭建 Blog 的过程，主要说过程，细节配置不详述。

GitHub Pages

参考 GitHub Pages 搭建教程 - 少数派 使用GitHub自带的主题先让页面能够通过自己的域名访问：

  1. 创建名为 username.GitHub.io 的仓库
  2. 到 Settings 的 GitHub Pages 里 Choose a theme
  3. 配上HTTPS，然后去做域名解析CNAME到 username.GitHub.io

Hugo

参考Hugo Quick Start 确保Hugo能在本地跑并产生public

  1. 去下一个Hugo

    • 地址：Hugo Releases
    • 下那个 windows 的压缩包，里面的 hugo.exe 就是可用的成品
    • 把 hugo.exe 丢到合适的路径，然后配置系统变量Path
    • hugo version 能显示就算🆗了

  2. 建框架hugo new site HugoBlog

  3. 挑个Hugo主题：Complete List | Hugo Themes

    • 把主题clone下来：git clone https://github.com/olOwOlo/hugo-theme-even themes/even

    • 这里要注意Hugo版本问题，看了issue才知道 even 的 CodeBlok 显示语言只支持hugo 0.59，再老的主题估计情况更糟糕。

  4. 把主题里的config.toml、content 到site的根目录

  5. 本地跑起来康康：hugo server

  6. 🆗了就生成网页：hugo --baseUrl="https://blog.dontjudge.cn" 静态网页的那些东西就都在public里了

本地public上传GitHub

public有了，怎么上传呢?

  1. 把 public 整个删掉
  2. 去GitHub Desktop 把 username.GitHub.io clone 下来，就clone到public，因为要空文件夹才能作为clone目标路径，所以前面要整个删掉
  3. 再生成一遍到public: hugo --baseUrl="https://blog.dontjudge.cn"
  4. commit push

完事了👌

后记

网上给出的方案都是命令行push到仓库,看不到实实在在的页面总是感觉不太能接受,所以用了看起来有点别扭的push姿势.
