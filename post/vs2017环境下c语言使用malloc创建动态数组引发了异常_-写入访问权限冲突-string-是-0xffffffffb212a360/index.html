问题描述： VS2017环境下，C语言使用malloc()创建动态数组，引发了异常: 写入访问权限冲突。 string 是 0xFFFFFFFFB212A360。 测试代码如下：

#include<stdio.h>
int main(void)
{
	const int str_max_size = 1024;
	int str_size = 0;
	char* string= (char*)malloc(str_max_size * sizeof(char));
	if (string == NULL)
	{
		printf("fail to malloc()!\n");
		return;
	}
	string[0] = '1';
	string[1] = '2';
	string[2] = '\0';
	printf("%s", string);

	return 0;
}

VS2017 报错截图： 分析过程： 这段代码只是测试功能使用，逻辑清晰，结构简单，讲道理不该出现任何问题。 VS调试过程中，出现写入访问权限冲突： - 第一直觉就是数组越界，所以代码才改成了对 string[0] 、string[1] 、string[2]的写入，测试写入情况。 - 但是如截图所示，还是报错，大眼瞪小眼盯着代码看了半天，还是没有任何问题，猜测可能是VS（MSVC）自身的问题，报错不完整。 - 将源码ftp至树莓派，使用gcc编译： ……好的，头文件不全。

解决方式： 添加头文件 #include<stdlib.h> 问题解决。

小结： visual studio 号称宇宙第一IDE，各项功能也是相当强大，但是有些奇怪的问题（MSVC历史包袱）还是令人有点头疼。 大一写C语言的时候都是两个头文件 stdio.h和stdlib.h，不管三七二十一先写再说，太久没写C语言，少写了一个头文件折腾了半天，只能怪学艺不精了。
