本文记录 Blog 使用的 Hugo 主题 https://github.com/olOwOlo/hugo-theme-even 的自定义配置。

Hugo 版本：0.59

even 版本：v4.0.0

1. 主题安装与应用

clone 到本地就算安装成功：

1

git clone https://github.com/olOwOlo/hugo-theme-even themes/even

将 themes\even\exampleSite\config.toml 文件复制到 Hugo site 的根目录下即可应用，主题是通过 config.toml 的属性 theme 配置生效的。

2. toml配置项

不作赘述，直接看配置文件

3. utterances 评论系统

  1. 安装 GitHub App：https://github.com/apps/utterances

  2. config.toml 里配置（even 主题本身带有它的配置项和具体实现，否则要自己嵌入代码）

    1
2
3

    [params.utterances]       # https://utteranc.es/
owner = "Deng-Jiajun"                        # Your GitHub ID
repo = "Deng-Jiajun.github.io"               # The repo to store comments

  3. 其他配置(（主题，issu名称等）在 themes\even\layouts\partials\comments.html 里

    1
2
3
4
5
6
7

    <script src="https://utteranc.es/client.js"
    repo="{{ .Site.Params.utterances.owner }}/{{ .Site.Params.utterances.repo }}"
    issue-term="url"
    theme="github-light"
    crossorigin="anonymous"
    async>
</script>

utterances 是依托于 repository 的 issues 建立的评论功能

在静态网站上的评论会以 repository 的issue 呈现，试了一下，应该是不能回复评论

5. 页面底部自定义

网页底部一排的 social link 看得我脑袋疼，config.toml 里都注释掉还会保留一个 RSS，修改 themes\even\layouts\partials\footer.html 可修改页面底部信息。

  • 删除 social link
    注释 <div class="social-links">... 代码块

  • 顺便修改了下面的 copyright 信息：
    注释 <span class="copyright-year">... 代码块，添加一段蹩脚代码：

    1

    <span class="copyright-and-others">Copyright &copy; 2020 <a href="https://github.com/Deng-Jiajun" style="color:#c05b4d">Deng</a></span>. All Rights Reserved

另外，对于 Hugo 和 主题 Even 的描述，要修改文字内容要到 themes\even\i18n\zh-CN.yaml 当然最好是把 i18n 复制一份到根目录，覆盖主题里的

6. 使用iconfont

页面是纯文字的，看了 even 的作者 olowolo的blog 发现它的有图标欸，好看，但是是私人分支，没有传上 GitHub，只好自己做。

这里使用 iconfont 嵌入图标

参考 iconfont说明 的 font-class引用 部分

  • 从项目获取css链接地址//at.alicdn.com/t/font_1150064_q0lnvx1hi7d.css
  • 按照这种格式 <link rel="stylesheet" type="text/css" href="http://at.alicdn.com/t/font_1150064_q0lnvx1hi7d.css">
  • 插入到 themes\even\layouts\partials\head.html

然后在需要插入图标的位置 <i class="iconfont icon-xxx"></i> 插入就行了

参考这篇 inconfont的使用 说用 symbol 最先进，咱也不知道。

7. 隐藏文章

主题只提供了在首页隐藏文章，而在归档处能看得见文章的设定。
但是存在需要把一些不希望在正常页面能看得见的文章（没营养的流水账）上传的需求。即又想上传，又不希望被看见的需求。

解决方案：
将需要隐藏的文章放入 content/xx/
此时生成的文章能够通过 baseURL/xx/ 访问，如果写了tags/categories，也能被识别。
